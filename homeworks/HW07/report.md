## 2. Protocol

Опишите ваш "честный" unsupervised-протокол.

- Препроцессинг: для числовых признаков использовался `SimpleImputer` (медиана) и `StandardScaler`. Для категориальных — `SimpleImputer` (most_frequent) и `OneHotEncoder`. Все этапы объединены в `ColumnTransformer` и `Pipeline`.
- Поиск гиперпараметров:
  - KMeans: перебор $k$ в диапазоне [2, 5].
  - Второй метод: для Dataset 02 использовался DBSCAN ($eps \in [0.3, 0.5, 0.8]$), для остальных — AgglomerativeClustering ($k \in [2, 3]$, `linkage` $\in$ ['ward', 'complete']).
  - Лучшая модель выбиралась по максимальному значению Silhouette Score.
- Метрики: использовались Silhouette Score, Davies-Bouldin Index и Calinski-Harabasz Score. Для DBSCAN при расчете метрик игнорировались шумовые точки (label -1).
- Визуализация: применялся PCA для снижения размерности до 2D с последующим построением scatter plot, окрашенного по меткам кластеров.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- KMeans "ломается" на данных с очень разной плотностью или сложной геометрией (как в Dataset 02), где он пытается навязать "сферические" границы.
- Иерархическая кластеризация хорошо показала себя на Dataset 01, совпав по результатам с KMeans, но работает медленнее на больших выборках.
- Сильнее всего на результат повлияло масштабирование (без StandardScaler кластеры искажались признаками с большими значениями) и обработка пропусков в Dataset 04.

### 5.2 Устойчивость (обязательно для одного датасета)

- Какую проверку устойчивости делали: 5 запусков KMeans на Dataset 01 с разными значениями `random_seed` (0-4) при фиксированном k=2.
- Что получилось: средний коэффициент ARI составил 1.0000.
- Вывод: кластеризация абсолютно устойчива. Это говорит о том, что структура данных в Dataset 01 настолько выражена, что начальная инициализация центроидов не влияет на итоговый результат.

### 5.3 Интерпретация кластеров

- Как вы интерпретировали кластеры: на Dataset 01 кластеры представляют собой две четкие группы пользователей/объектов, разнесенные в пространстве главных компонент. В Dataset 04 кластеры, вероятно, соответствуют различным профилям поведения, определяемым комбинацией категориальных настроек и средних уровней числовых метрик.
- Вывод: кластеры имеют высокую плотность внутри и хорошее разделение между собой, что позволяет использовать их как новую категориальную переменную для дальнейшего обучения с учителем.

## 6. Conclusion

1. Стандартизация — критический этап для метрических алгоритмов кластеризации.
2. Метрика Silhouette хорошо подходит для выбора количества кластеров, но требует визуального подтверждения через PCA.
3. Обработка пропусков и категориальных признаков существенно усложняет pipeline, но необходима для реальных данных.
4. KMeans остается самым стабильным и быстрым базовым алгоритмом для данных со сферической структурой.
5. Индекс ARI позволяет количественно оценить надежность полученного разбиения.
6. Выбор алгоритма сильно зависит от геометрии данных: DBSCAN для плотности, KMeans для "центров".
