## 3. Эксперименты

### 3.1. Часть A (S10): классификация, аугментации и transfer learning (обязательно)

Задача: показать эффект CNN, аугментаций и transfer learning.

Провести 4 эксперимента (C1-C4). Эксперимент = фиксированный конфиг модели/обучения + обучение + оценка на `val`.

- **C1 (simple-cnn-base)**: простая CNN без аугментаций.
- **C2 (simple-cnn-aug)**: та же CNN, но с разумными аугментациями.
- **C3 (resnet18-head-only)**: `ResNet18` с pretrained weights; backbone заморожен, обучается только классификационная голова.
- **C4 (resnet18-finetune)**: `ResNet18` с pretrained weights; частичное fine-tune (например, `layer4 + fc`, либо другой близкий по смыслу вариант).

Требования:

- loss: `CrossEntropyLoss`;
- основная метрика: accuracy;
- сравнение должно быть корректным: фиксированный split, фиксированный seed, сопоставимые условия обучения;
- лучшая модель части A выбирается по `best_val_accuracy`;
- test-часть используется **один раз**: только для финальной оценки лучшей модели после выбора по `val`.

Важно:

- в C1 и C2 архитектура CNN должна быть одной и той же, меняются только аугментации;
- в C3 и C4 обязательно использовать именно pretrained `ResNet18` из `torchvision.models`;
- в отчёте нужно явно объяснить, что дали аугментации и что дало partial fine-tuning относительно head-only.

### 3.2. Часть B (S11): детекция **или** сегментация (обязательно)

Задача: показать, что вы умеете запускать готовый CV-пайплайн, визуализировать результат и считать корректную базовую метрику.

Нужно выбрать **один трек**.

#### Вариант 1: Detection track

Используйте готовую pretrained detection-модель (например, `FasterRCNN_ResNet50_FPN` или другой близкий по смыслу вариант из `torchvision`).

Проведите 2 режима инференса:

- **V1**: inference при `score_threshold = 0.3`
- **V2**: inference при `score_threshold = 0.7`

Для каждого режима:

- визуализируйте предсказанные bounding boxes на нескольких изображениях;
- реализуйте базовое сопоставление prediction ↔ ground truth при `IoU >= 0.5`;
- посчитайте минимум:
  - `precision`
  - `recall`
  - `mean_iou` по сопоставленным предсказаниям

Требования:

- считать **полный mAP не требуется**;
- важно корректно объяснить, как изменяются `precision` и `recall` при изменении порога уверенности.

#### Вариант 2: Segmentation track

Используйте готовую pretrained segmentation-модель (например, `DeepLabV3_ResNet50`, `FCN_ResNet50` или другой близкий по смыслу вариант из `torchvision`).

Проведите 2 режима оценки:

- **V1**: базовая постобработка маски
- **V2**: альтернативная постобработка маски

В качестве альтернативной постобработки можно выбрать **один** из вариантов:

- другой порог для бинаризации выбранного foreground-класса;
- удаление слишком маленьких компонент связности;
- простую морфологическую очистку маски;
- иной разумный и воспроизводимый способ, который вы кратко опишете.

Для каждого режима:

- визуализируйте предсказанную маску на нескольких изображениях;
- посчитайте минимум:
  - `mean_iou`
- дополнительно приветствуется:
  - `pixel_precision`
  - `pixel_recall`

Требования:

- не требуется делать multi-class benchmark;
- достаточно аккуратно рассмотреть один понятный foreground-сценарий;
- в отчёте нужно объяснить, что именно вы считали foreground и почему.

Важно:

- во второй части не нужно сохранять веса модели;
- во второй части важны корректная визуализация, постановка метрики и интерпретация результата.
