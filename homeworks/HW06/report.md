## 1. Dataset

- Какой датасет выбран: `S06-hw-dataset-04.csv`
- Размер: 25 000 строк, 62 столбца (60 признаков + `id` + `target`)
- Целевая переменная: `target` (бинарная классификация, около 95% объектов относятся к классу 0 — сильный дисбаланс)
- Признаки: только числовые признаки (`f01`–`f60`), столбец `id` исключен из обучения.

## 2. Protocol

- Разбиение: train/test в пропорции 75/25, с фиксацией `random_state=42` и использованием `stratify=y` для сохранения баланса классов.
- Подбор: кросс-валидация на 5 фолдов (`StratifiedKFold`) исключительно на обучающей выборке. Оптимизировалась метрика `roc_auc`.
- Метрики:
  - `accuracy`: базовый показатель (здесь менее информативен из-за дисбаланса).
  - `f1`: оценка качества нахождения редкого класса.
  - `roc_auc`: основная метрика ранжирования вероятностей, устойчивая к дисбалансу.

## 3. Models

В ходе эксперимента сравнивались следующие модели:

- **DummyClassifier (baseline)**: стратегия `most_frequent`. Показал высокую точность (95%), но нулевой F1, так как не нашел ни одного объекта редкого класса.
- **LogisticRegression (baseline)**: использовался пайплайн со `StandardScaler`. Подбирался параметр регуляризации `C`.
- **DecisionTreeClassifier**: контроль сложности через `max_depth`, `min_samples_leaf` и `ccp_alpha` (cost-complexity pruning).
- **RandomForestClassifier**: ансамбль (bagging). Подбирались глубина и минимальное число объектов в листе.
- **Boosting (HistGradientBoostingClassifier)**: градиентный бустинг. Подбирались `learning_rate` и `max_depth`.
- **StackingClassifier**: композиция из всех вышеперечисленных моделей (кроме Dummy) с логистической регрессией в качестве мета-модели.

## 4. Results

Итоговые метрики на тестовой выборке:

| Модель | ROC-AUC | Accuracy | F1-score |
| :--- | :--- | :--- | :--- |
| **Stacking** | **0.9079** | 0.9803 | 0.7602 |
| Boosting | 0.9066 | 0.9774 | 0.7069 |
| RandomForest | 0.9004 | 0.9720 | 0.6032 |
| LogisticRegression | 0.8400 | 0.9626 | 0.4091 |
| DecisionTree | 0.8022 | 0.9675 | 0.5653 |
| Dummy | 0.5000 | 0.9509 | 0.0000 |

**Победитель**: `StackingClassifier`. Модель показала лучший ROC-AUC и F1, за счет комбинирования сильных сторон линейных моделей и ансамблей деревьев.

## 5. Analysis

- **Устойчивость**: При изменении `random_state` метрики ансамблей (RF, Boosting, Stacking) колеблются в пределах ±0.005–0.01 по ROC-AUC, что говорит о высокой стабильности решений по сравнению с одиночным деревом.
- **Ошибки**: Confusion matrix для лучшей модели показывает, что стекинг успешно находит большинство объектов редкого класса, минимизируя ложноположительные срабатывания (FP).
- **Интерпретация**: Permutation importance выявил около 5 ключевых признаков среди `f01-f60`, которые вносят основной вклад в разделение классов. Случайное перемешивание этих признаков приводит к резкому падению ROC-AUC.

## 6. Conclusion

1. Одиночные деревья решений крайне склонны к переобучению, если не использовать параметры контроля сложности (`ccp_alpha`, `max_depth`).
2. Ансамбли (Random Forest и градиентный бустинг) значительно превосходят базовые модели на сложных данных с дисбалансом.
3. Стекинг позволяет выжать максимум из имеющихся моделей, эффективно объединяя их предсказания.
4. "Честный" протокол (фиксация seed, стратификация, CV только на train) критически важен: без него легко получить завышенные метрики, которые не подтвердятся на реальных данных.
5. При сильном дисбалансе классов метрика Accuracy может вводить в заблуждение, поэтому ROC-AUC и F1 являются обязательными для объективной оценки.
